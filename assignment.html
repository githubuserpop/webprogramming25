Vue 3 Web Development Assignment

Assignment Title:

Build Your Own Personal Website with Vue 3

Objective:

Students will create a personal website using Vue 3 that includes multiple sections and interactive elements. This project will cover key Vue concepts such as:

  • ref() for reactivity
  • v-model for two-way data binding
  • v-for for list rendering
  • v-if and v-show for conditional rendering
  • v-on for event handling
  • computed() for derived data
  • watch() for tracking changes
  • Component-based structure

This assignment will serve as the foundation for your next project, so organize your code well and keep it easily extendable.

Project Requirements (40 Points Total)

1. Setup Your Vue Project (5 points)

  • Create a new Vue 3 project using Vite.
  • Organize your files properly (use src/components/ for reusable components).

2. Website Structure (10 points)

Your website should contain at least three sections that can be navigated within a single-page layout (e.g., using buttons or links to scroll between sections):

  • Home Section: Introduce yourself (Use v-if to conditionally display a welcome message).
  • Projects Section: Display a list of projects using v-for.
  • Contact Section: Include a form where users can submit their contact details using v-model.

3. Interactivity and State Management (10 points)

  • Include a counter somewhere on the website (e.g., number of visitors, button clicks).
  • The counter should be reactive using ref() and update when a button is clicked (v-on).
  • Add a message toggle button that shows or hides extra information (Use v-show to control visibility).

4. Dynamic Data with Computed and Watch (5 points)

  • Use computed() to display a summary of data (e.g., number of projects).
  • Use watch() to detect changes in form inputs and log them to the console.

5. Conditional Rendering and Styling (5 points)

  • Use v-if to show a "No projects available" message if the project list is empty.
  • Use v-show to hide/display extra information when a button is clicked.

6. Documentation & Maintainability (5 points)

  • Submit a short project documentation file (README.md) that includes:
    • A brief project description.
    • Instructions on how to set up and run the project.
    • Any additional notes or challenges faced during development.
  • Keep the project structured and extendable for future assignments.

Bonus (10 Points, Optional)

  • (5 points) Store form submissions in localStorage and persist them on page refresh.
  • (5 points) Add animations (e.g., smooth transitions, fade-in effects for section changes).

Submission Guidelines

  • Submit a GitHub repository link with your complete project.
  • Ensure your project is fully functional and responsive.
  • Include a README.md file with project details.
  • Keep your code organized, as this project will be expanded upon in future assignments.
